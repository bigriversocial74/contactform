# Training Lab Pricing Concept

## Purpose

This document defines the pricing concept for the future Training Lab by Microgifter public pricing page and commercial model.

This is a planning document only. It does not approve implementation, billing code, checkout code, subscription logic, payment processing, or production pricing changes.

## Pricing positioning

Training Lab pricing should be simple, minimal, and easy to understand.

The pricing page should sell the outcome:

```text
Simple pricing for action-based training rewards.
```

The page should explain that Training Lab helps organizations launch proof-based training, track consistency, and run reward campaigns that keep people engaged.

## Design direction

The pricing page should follow the approved Training Lab marketing style:

```text
minimal white layout
dark green primary CTAs
soft mint highlights
clean line-art illustration
simple cards
story-first copy
no dense feature matrix at the top
```

The pricing page should not feel like a crowded SaaS comparison table. It should help visitors choose the right level quickly.

## Pricing tiers

## Free

For individuals testing proof-based training.

Suggested price:

```text
$0 forever
```

Primary use case:

```text
Try Training Lab with one basic campaign and simple proof uploads.
```

Suggested included features:

```text
1 active campaign
up to 10 participants
proof uploads
basic streak tracking
starter reward tracking
```

Primary CTA:

```text
Get Started Free
```

## Team

For small teams building healthy habits and consistent action.

Suggested price:

```text
$9 / user / month
```

Primary use case:

```text
Run multiple team challenges, track proof, and reward consistent participation.
```

Suggested included features:

```text
unlimited campaigns
up to 100 participants
proof uploads and action receipts
streaks and milestones
email reminders
team progress views
```

Primary CTA:

```text
Start Team Plan
```

## Organization

For growing teams running reward programs.

Suggested price:

```text
$19 / user / month
```

This should be the highlighted plan.

Primary use case:

```text
Run structured proof-based training and reward programs across larger groups.
```

Suggested included features:

```text
everything in Team
unlimited participants
advanced reward rules
custom branding
admin review queue
action receipts
priority support
```

Primary CTA:

```text
Start Organization Plan
```

## Enterprise

For large organizations with advanced needs.

Suggested price:

```text
Custom
```

Primary use case:

```text
Custom proof, training, rewards, integrations, permissions, and high-volume campaign needs.
```

Suggested included features:

```text
everything in Organization
SSO / SAML
advanced integrations
dedicated success manager
custom contract and SLA
custom reward programs
custom reporting
```

Primary CTA:

```text
Contact Sales
```

## Pricing page structure

Recommended public pricing page layout:

```text
1. Header / navigation
2. Pricing hero
3. Monthly / annual toggle
4. Four pricing cards
5. Every plan includes section
6. FAQ
7. Final CTA
8. Minimal footer
```

## Hero copy

Suggested headline:

```text
Simple pricing for action-based training rewards.
```

Suggested subheadline:

```text
Launch proof-based training, track consistency, and run reward campaigns that keep people engaged.
```

## Annual billing concept

The pricing page can include a simple monthly / annual toggle.

Suggested annual discount language:

```text
Save up to 20%
```

The annual toggle is a visual and planning concept only until billing logic is approved.

## Every plan includes

The pricing page should use a small visual band below the pricing cards.

Suggested items:

```text
Proof Uploads
Action Receipts
Progress Tracking
Reward Rules
```

Descriptions:

```text
Proof Uploads: Add photos, videos, or documents as proof.
Action Receipts: Capture and verify completed training actions.
Progress Tracking: Track streaks, milestones, and consistency.
Reward Rules: Automate rewards based on verified actions.
```

## FAQ topics

Suggested FAQ questions:

```text
Can I change plans later?
Is there a limit on proof uploads?
Do you offer discounts for annual billing?
Can rewards be customized?
Can we use this for teams, schools, wellness groups, or creators?
```

## Commerce connection

Pricing should connect cleanly to the future commerce pages:

```text
Pricing page -> Cart -> Checkout -> Checkout Success -> Receipt
```

The first commerce flow should stay simple:

```text
select plan
review cart
enter billing/contact information
complete purchase
show success confirmation
show receipt
```

## Cart concept

The cart page should show selected plan or add-ons in a clean order summary.

Suggested cart items:

```text
Team Plan
Organization Plan
Extra Proof Uploads
Additional Participant Seats
```

The cart should support:

```text
quantity / user count
annual discount display
promo code area
request quote action
continue checkout CTA
```

## Checkout concept

The checkout page should be clean and trust-focused.

Suggested sections:

```text
billing and contact information
payment method
order summary
secure checkout note
complete purchase CTA
```

The checkout UI should not include real payment integrations until implementation is explicitly approved.

## Notes for future implementation

Do not implement billing, subscriptions, payments, coupons, plan enforcement, or checkout processing until the user explicitly approves code implementation.

Future implementation should treat these pricing tiers as draft product strategy until the business model and production billing rules are confirmed.
