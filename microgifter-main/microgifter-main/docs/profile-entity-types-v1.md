# Profile Entity Types v1

## Purpose

Future Demand Profiles must support more than traditional merchants. A profile can represent any entity that can attract, receive, stimulate, or organize future demand.

## Entity types

Primary profile entity types:

- person
- creator
- artist
- musician
- business
- merchant
- brand
- organization
- nonprofit
- venue
- event host
- team
- community group
- enterprise sponsor
- regional program

## Shared fields

Every profile type should eventually support:

- public handle
- display name
- owner user ID
- profile type
- category
- region
- visibility
- verification status
- active status
- contact and interaction options
- demand signal summary

## Profile roles

A profile can participate in more than one role:

- attracts demand
- sells products or services
- publishes digital assets
- receives gifts or support
- sends gifts or rewards
- sponsors local demand
- organizes regional or category activity
- creates content
- hosts locations or events

## Examples

A musician can attract future demand through shows, merch, fan gifting, content, subscriptions, and local venue activity.

A restaurant can attract future demand through pre-purchased meals, giftable offers, reviews, redemptions, locations, and workplace rewards.

An enterprise sponsor can stimulate future demand through employee rewards, customer incentives, local purchase commitments, and regional programs.

A nonprofit can attract and stimulate demand through donations, event participation, content, volunteer activity, and local partner programs.

## Stage boundary

Stage 6C defines the language. Stage 15 and Stage 16 should introduce the scoring, intelligence, and agent recommendation systems.
